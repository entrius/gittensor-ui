# Smart Contract Metadata

This folder contains the ABI/metadata JSON files required by `@polkadot/api-contract` to interact with deployed ink! WASM smart contracts on Substrate.

## Files

### IssueBountyManager.json

The contract metadata for the Issue Bounty System smart contract. This file is generated by the ink! compiler (`cargo contract build`) and contains:

- **ABI**: Function signatures, events, and storage layout
- **WASM**: Compiled WebAssembly bytecode (embedded)
- **Types**: Substrate type definitions for encoding/decoding

**Source**: Generated from `/home/landyn/dev/issues-smart-contract/`

**Usage**: Imported by `IssueRegistrationModal.tsx` to create a `ContractPromise` instance for calling contract methods like `register()`.

## Updating Contract Metadata

When the smart contract is recompiled:

1. Build the contract: `cargo contract build --release`
2. Copy the generated `.json` file from `target/ink/` to this folder
3. Restart the dev server to pick up changes
